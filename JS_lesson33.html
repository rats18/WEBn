1. 학습 목표

  • 배열과 반복문을 활용하여 여러 링크를 효율적으로 제어

  • 프로젝트 완성도를 높이는 고급 기능 구현

2. querySelectorAll로 다중 요소 선택

2.1 기본 사용법

var alist = document.querySelectorAll('a');

console.log(alist.length); // 전체 링크 개수 확인

2.2 querySelector vs querySelectorAll

  • querySelector(): 첫 번째 요소만 반환
  • querySelectorAll(): 모든 해당 요소를 NodeList로 반환
  • NodeList는 배열처럼 인덱스와 length 속성 사용 가능

3. 반복문으로 모든 링크 제어

3.1 기본 반복 구조

var alist = document.querySelectorAll('a');

var i = 0;

while (i < alist.length) {

alist[i].style.color = 'powderblue';

i = i + 1;

}

3.2 Night & Day 모드 통합

Night 모드 링크 처리:

function nightMode(self) {
    var target = document.querySelector('body');
    var alist = document.querySelectorAll('a');
    var i = 0;
    
    if (self.value === 'night') {
        target.style.backgroundColor = 'black';
        target.style.color = 'white';
        self.value = 'day';
        
        while (i < alist.length) {
            alist[i].style.color = 'powderblue';
            i = i + 1;
        }
    } else {
        target.style.backgroundColor = 'white';
        target.style.color = 'black';
        self.value = 'night';
        
        while (i < alist.length) {
            alist[i].style.color = 'blue';
            i = i + 1;
        }
    }
}

5. 배열 활용의 핵심 가치

5.1 확장성 (Scalability)

  • 링크 개수가 늘어나도 코드 수정 불필요
  • 1개든 1억개든 동일한 코드로 처리
  • 새로운 링크 추가 시 자동으로 적용

5.2 유지보수성 (Maintainability)

  • 중복 코드 제거로 버그 발생 가능성 감소
  • 스타일 변경 시 한 곳만 수정하면 모든 요소에 적용
  • 코드 구조가 명확하고 이해하기 쉬움

5.3 성능 최적화

  • DOM 선택을 한 번만 수행
  • 효율적인 반복 처리
  • 브라우저 리소스 절약

6. CSS 선택자 활용 확장

6.1 다양한 선택자 패턴

document.querySelectorAll('a')              // 모든 a 태그

document.querySelectorAll('.nav-link')      // 클래스 선택

document.querySelectorAll('nav a')           // 자손 선택

document.querySelectorAll('a[href^="http"]')  // 속성 선택

6.2 정밀한 타겟팅

  • CSS 선택자 지식을 JavaScript에서 활용
  • 원하는 요소들만 정확히 선택
  • 복잡한 선택자도 자유롭게 사용

7. 실무 적용 패턴

7.1 테마 시스템

  • 다크 모드 / 라이트 모드 전환
  • 모든 UI 요소 일괄 스타일링
  • 사용자 설정 저장 및 복원

7.2 동적 메뉴 관리

  • 네비게이션 메뉴 활성화 상태 관리
  • 메뉴 항목 숨김/표시 제어
  • 반응형 메뉴 동작 구현

7.3 대량 데이터 처리

  • 게시물 목록 일괄 스타일링
  • 상품 카드 디자인 변경
  • 테이블 행/열 조작

8. 성능 고려사항

8.1 효율적인 DOM 접근

// 좋은 방법: 변수에 저장하여 재사용

var links = document.querySelectorAll('a');

while (i < links.length) { ... }


// 나쁜 방법: 매번 DOM 탐색

while (i < document.querySelectorAll('a').length) { ... }

8.2 반복문 최적화

  • 필요한 요소만 선택하여 불필요한 작업 방지
  • 반복문 내부에서 복잡한 연산 최소화
  • 가능한 경우 CSS 클래스 활용
