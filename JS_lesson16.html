1. 함수 기본 개념

  • 영어로 Function
  • 반복되지만 연속적이지 않은 코드를 정리하는 도구
  • 코드를 정리정돈하기 위한 수납상자

2. 함수가 필요한 상황

2.1 반복문으로 해결 안되는 경우

document.write("2<br>");

document.write("3<br>");

// 다른 코드들...

document.write("2<br>");  // 같은 코드가

document.write("3<br>");  // 다시 등장

  • 반복문 사용 불가: 연속적이지 않은 중복
  • 1억개 중복 시: 반복문으로는 해결 불가능

3. 함수 기본 문법

3.1 함수 정의 (Function Declaration)

function two() {

    document.write("2<br>");

    document.write("3<br>");

}

3.2 함수 호출 (Function Call)

two();  // 함수 실행

3.3 완성된 코드

document.write("1<br>");

two();                    // 함수 호출

document.write("4<br>");

two();                    // 함수 재호출

document.write("5<br>");

4.1 함수 정의와 호출

4.2 함수 재사용

4.3 버튼 색상 변경 함수

5. 함수 동작 원리

5.1 웹브라우저의 처리 과정

  1. 함수 정의 인식: function two() { ... } 발견
  2. 함수 내용 기억: 중괄호 안의 코드를 'two'라는 이름으로 저장
  3. 함수 호출 인식: two() 발견 시 괄호로 실행 명령임을 인식
  4. 함수 실행: 저장된 코드를 해당 위치에서 실행

5.2 실행 순서 분석

// 실행 순서:

document.write("1<br>");     // 1 출력

two();                       // 2, 3 출력 (함수 실행)

document.write("4<br>");     // 4 출력

two();                       // 2, 3 출력 (함수 재실행)

document.write("5<br>");     // 5 출력

// 결과: 1, 2, 3, 4, 2, 3, 5

6. 함수 구성 요소

6.1 필수 구성 요소

  • function 키워드: 함수 정의 선언
  • 함수명: 재사용할 때 호출할 이름
  • () 괄호: 매개변수 자리 (지금은 비워둠)
  • {} 중괄호: 실행할 코드 블록
  • 함수 호출: 함수명() 형태로 실행

6.2 문법 구조

function 함수명() {

    // 재사용할 코드

}

// 사용법

함수명();  // 함수 실행

7. 함수의 핵심 가치

7.1 코드 재사용성

  • 같은 코드 반복 작성 불필요
  • 함수명으로 간단한 호출

7.2 유지보수성

  • 한 곳만 수정하면 모든 사용처 반영
  • 코드 변경이 안전하고 확실

7.3 가독성

  • 복잡한 코드를 의미있는 이름으로 추상화
  • 전체 코드 흐름 이해 용이

8. 함수와 반복문의 차이

  • 반복문: 연속적인 반복 작업
  • 함수: 연속적이지 않은 중복 코드 정리
  • 활용 시점: 같은 코드가 여러 곳에 흩어져 있을 때

9. 핵심 기억사항

  1. function 키워드: 함수 정의의 시작
  2. 함수명(): 정의된 함수를 실행하는 방법
  3. 코드 정리정돈: 복잡한 로직을 간단한 이름으로 포장
  4. 재사용성: 한 번 정의하면 몇 번이든 호출 가능

10. 함수의 본질

함수는 단순한 문법이 아니라 복잡한 소프트웨어를 관리 가능하게 만드는 핵심 도구. 코드 복잡도가 증가할 때 정말 엄청난 이득을 주는 정리정돈 도구.
