1. 리팩토링이란?

  • Re-factory: 공장으로 다시 보내서 개선
  • 동작은 그대로 두고 코드를 효율적으로 개선
  • 가독성 향상, 유지보수 편의, 중복 제거

2. 현재 코드의 문제점

2.1 중복 코드

Night & Day 토글 기능에서 document.querySelector('body')가 여러 번 반복됨

2.2 긴 onclick 속성

onclick 안에 모든 코드가 들어가서 읽기 어려움

2.3 유지보수의 어려움

색상을 변경하려면 여러 곳을 일일이 찾아서 수정해야 함

3. 극단적 상황 가정

3.1 버튼이 1억개라면?

  • 배경색을 black에서 navy로 변경할 때
  • 1억개의 onclick 코드를 모두 수정해야 함
  • 실수 위험: 일부는 변경되고 일부는 그대로
  • 웹페이지 용량 증가: 중복 코드로 인한 크기 폭증

3.2 현실적 문제

  • 시간이 많이 걸림
  • 자주 반복됨
  • 실수가 자주 발생
  • 불행과 절망감 증가

4. 리팩토링의 목표

4.1 중복 제거

같은 코드가 여러 번 나오는 것을 하나로 통합

4.2 코드 정리정돈

복잡한 코드를 간단하고 읽기 쉽게 만들기

4.3 유지보수성 향상

한 곳만 수정하면 모든 곳에 적용되도록

5. 리팩토링의 핵심 원칙

5.1 동작 보존

기능은 정확히 같게 유지하면서 코드만 개선

5.2 점진적 개선

한 번에 모든 것을 바꾸지 말고 단계적으로 진행

6. 코딩 잘하는 방법의 비밀

"중복을 끝까지 쫓아가서 다 없애 버리세요"

대부분의 프로그래밍 기술이 중복 제거에서 출발함:

  • 변수: 중복되는 값을 하나로
  • 조건문: 중복되는 코드 흐름을 정리
  • 앞으로 배울 도구들도 모두 중복 제거가 목적

7. 리팩토링 후 기대 효과

  • 코드 양 감소: 간결하고 읽기 쉬운 구조
  • 유지보수성 향상: 수정이 쉽고 안전
  • 확장성 증대: 새로운 기능 추가 용이
  • 버그 감소: 중복으로 인한 실수 방지
