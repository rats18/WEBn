1. 프로퍼티와 메서드란?

  • 프로퍼티(Property): 객체에 저장된 데이터(변수)
  • 메서드(Method): 객체에 저장된 함수
  • 같은 것이지만 객체에 속했을 때 부르는 이름이 다름

2. 기본 개념 정리

2.1 독립적일 때 vs 객체에 속할 때

  • 독립된 데이터 → 변수(Variable)
  • 객체 안의 데이터 → 프로퍼티(Property)
  • 독립된 함수 → 함수(Function)
  • 객체 안의 함수 → 메서드(Method)

2.2 실제 예시

// 독립적인 변수와 함수

var name = "김철수";

function greeting() {

    alert("안녕하세요!");

}

// 객체의 프로퍼티와 메서드

var person = {

    name: "김철수",            // 프로퍼티

    greeting: function() {      // 메서드

        alert("안녕하세요!");

    }

};

3. 메서드 정의와 호출

3.1 메서드 정의 문법

var myObject = {

    methodName: function() {

        // 실행할 코드

    }

};

3.2 메서드 호출 문법

myObject.methodName();  // 점 표기법으로 호출

4. this 키워드 이해

4.1 this란?

  • this: 메서드가 속한 객체를 가리키는 키워드
  • 자기참조: 객체가 자기 자신을 참조하는 방법
  • 동적 결정: 호출되는 순간에 어떤 객체인지 결정

4.2 this 사용 예제

4.3 this의 핵심 이유

var student1 = {

    name: "김학생",

    study: function() {

        alert(this.name + "이 공부합니다");

    }

};

var student2 = {

    name: "이학생",

    study: function() {

        alert(this.name + "이 공부합니다");

    }

};

// this 덕분에 같은 메서드가 다른 객체에서 다르게 동작

6. 메서드 체이닝

6.1 메서드 체이닝이란?

  • 메서드 연결: 여러 메서드를 연속으로 호출
  • return this: 메서드에서 객체 자신을 반환
  • 가독성: 연속적인 작업을 한 줄로 표현

7. 객체 사용의 장점

7.1 정리정돈

  • 관련 데이터와 함수 그룹핑: 하나의 객체에 모든 관련 기능 집중
  • 네임스페이스 효과: 이름 충돌 방지
  • 구조화: 코드의 논리적 구조 명확화

7.2 재사용성

  • 템플릿 역할: 같은 구조의 여러 객체 생성 가능
  • 모듈화: 독립적인 기능 단위로 분리
  • 확장성: 새로운 프로퍼티와 메서드 추가 용이

10. 핵심 개념 정리

10.1 용어 정리

  1. 프로퍼티: 객체의 데이터 (키: 값)
  2. 메서드: 객체의 함수 (키: function)
  3. this: 메서드 내에서 객체 자신을 참조
  4. 메서드 체이닝: return this로 연속 호출

10.2 문법 패턴

var 객체명 = {

    프로퍼티명: 값,

    메서드명: function() {

        // this로 자기 참조

        return this; // 체이닝용

    }

};

11. 실무 활용 팁

11.1 설계 원칙

  • 관련성: 서로 관련된 데이터와 기능만 하나의 객체에
  • 일관성: 비슷한 기능은 비슷한 이름으로
  • 단순성: 너무 복잡한 객체보다는 단순하고 명확하게

11.2 네이밍 컨벤션

  • camelCase: firstName, lastName, getUserInfo
  • 동사 + 명사: setColor, getName, calculateTotal
  • 의미 있는 이름: data1보다는 userInfo가 좋음
