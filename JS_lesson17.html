1. 목표: 리팩토링을 통한 코드 효율화

  • 리팩토링: 동작은 같지만 코드를 효율적으로 개선
  • 함수: 리팩토링의 핵심 도구

2. 함수 구현 과정

2.1 함수 정의

function nightDayHandler(self) {

    var target = document.querySelector('body');

    if(self.value === 'night') {

        target.style.backgroundColor = 'black';

        target.style.color = 'white';

        self.value = 'day';

    } else {

        target.style.backgroundColor = 'white';

        target.style.color = 'black';

        self.value = 'night';

    }

}

2.2 함수 호출

<input type="button" value="night" onclick="nightDayHandler(this)">

3. this와 매개변수의 관계

3.1 문제 상황

// 독립된 함수에서 this는 전역객체(window)를 가리킴

function nightDayHandler() {

    // 여기서 this는 input 버튼이 아님!

}

3.2 해결책: 매개변수 활용

function nightDayHandler(self) {

    // self = this 값을 매개변수로 전달받음

    if(self.value === 'night') {

        // self를 통해 input 버튼 제어

    }

}

3.3 호출 시 this 전달

<input onclick="nightDayHandler(this)">

5. 함수 사용의 효과

5.1 코드 중복 제거

// 함수 사용 전: 중복 코드

<input onclick="복잡한 JavaScript 코드">

<input onclick="복잡한 JavaScript 코드">

// 함수 사용 후: 간결한 호출

<input onclick="nightDayHandler(this)">

<input onclick="nightDayHandler(this)">

5.2 유지보수성 향상

  • 한 곳만 수정: 함수 내용 변경 시 모든 사용처에 적용
  • 1억개 버튼: 동일한 코드로 모두 관리 가능

5.3 가독성 개선

  • 의미 명확화: nightDayHandler라는 이름으로 기능 파악
  • 코드 간소화: 복잡한 로직을 간단한 호출로 대체

function nightDayHandler(self) {

    // self는 클릭된 input 요소

}

7. 리팩토링의 핵심 가치

7.1 확장성

  • 버튼 추가 시 단순 복사만으로 기능 구현
  • 새로운 기능 추가 시 함수만 수정

7.2 안정성

  • 모든 버튼이 동일한 로직으로 동작
  • 실수로 인한 불일치 방지

7.3 효율성

  • 코드 양 감소
  • 네트워크 전송 비용 절약

8. 비교: 함수 사용 전 vs 후

8.1 함수 사용 전 (13강)

onclick="if(this.value === 'night') { 긴 코드... } else { 긴 코드... }"

  • onclick 속성이 매우 길어짐
  • 중복 코드 반복
  • 수정 시 모든 곳을 찾아서 변경

8.2 함수 사용 후 (17강)

onclick="nightDayHandler(this)"

  • onclick 속성이 간결함
  • 함수 한 곳에서 모든 로직 관리
  • 수정 시 함수만 변경하면 모든 곳에 적용

9. 함수의 본질적 가치

"복잡한 코드를 의미있는 이름으로 포장하여, 재사용 가능하고 관리하기 쉬운 단위로 만드는 것"

함수를 통해 코드가 더욱 체계적이고 효율적으로 관리될 수 있습니다!

10. 리팩토링 단계 완성

14강에서 리팩토링의 필요성을 인식하고, 15강에서 중복 제거 기법을 학습하고, 16강에서 함수 기본 문법을 익힌 후, 17강에서 실제 프로젝트에 함수를 적용하여 리팩토링 단계를 완전히 마무리했습니다.

11. 핵심 기억사항

  1. 매개변수: this를 self로 전달받아 함수 내에서 활용
  2. 함수명: nightDayHandler처럼 기능을 명확히 표현
  3. 코드 분리: HTML에서 복잡한 로직을 함수로 분리
  4. 재사용성: 한 번 정의한 함수를 여러 곳에서 호출
