2. 모든 a 태그 선택하기

2.1 document.querySelectorAll() 사용

var alist = document.querySelectorAll('a');

  • querySelector(): 첫 번째 요소만 선택
  • querySelectorAll(): 모든 해당 요소를 배열 형태로 선택

2.2 선택 결과 확인

console.log(alist[0]);      // 첫 번째 a 태그

console.log(alist[1]);      // 두 번째 a 태그

console.log(alist.length);  // a 태그 총 개수

3. 반복문으로 모든 링크 제어

3.1 기본 반복문 구조

var alist = document.querySelectorAll('a');

var i = 0;

while (i < alist.length) {

    alist[i].style.color = 'powderblue';

    i = i + 1;

}

3.2 완성된 Night & Day 기능

// Night 모드

var alist = document.querySelectorAll('a');

var i = 0;

while (i < alist.length) {

    alist[i].style.color = 'powderblue';

    i = i + 1;

}

// Day 모드

var alist = document.querySelectorAll('a');

var i = 0;

while (i < alist.length) {

    alist[i].style.color = 'blue';

    i = i + 1;

}

5. 핵심 원리

5.1 검색 지향 프로그래밍

  • 검색 능력이 프로그래밍의 핵심
  • 필요한 기능을 검색으로 찾아서 활용
  • 예: "javascript get element by css selector multiple"

5.2 배열과 반복문의 조합

  1. querySelectorAll(): 여러 요소를 배열로 가져오기
  2. while문: 배열의 각 요소를 순차적으로 처리
  3. 동적 길이 활용: .length로 배열 크기에 맞춰 반복

6. 실제 동작 과정

6.1 Night 버튼 클릭 시

  1. 모든 a 태그를 배열로 선택
  2. 반복문으로 각 태그의 색상을 powderblue로 변경
  3. 어두운 배경에서 링크가 잘 보이도록 개선

6.2 Day 버튼 클릭 시

  1. 모든 a 태그를 배열로 선택
  2. 반복문으로 각 태그의 색상을 blue로 변경
  3. 밝은 배경에서 링크가 선명하게 보이도록 복원

7. 반복문의 위력

7.1 확장성

  • 링크가 1개든 1억개든 동일한 코드로 처리
  • 새로운 링크 추가 시 자동으로 적용

7.2 유지보수성

  • 색상 변경: 한 곳만 수정하면 모든 링크 일괄 적용
  • 코드 간소화: 반복 작업을 몇 줄로 처리

9. CSS 선택자 활용

9.1 다양한 선택자

document.querySelectorAll('a')              // 모든 a 태그

document.querySelectorAll('.link')          // 클래스가 link인 모든 요소

document.querySelectorAll('input[type=button]')  // 모든 버튼

document.querySelectorAll('h1, h2, h3')        // 모든 제목 태그

9.2 정확한 타겟팅

  • CSS 선택자 지식이 JavaScript에서도 활용
  • 원하는 요소들만 정확히 선택 가능
  • 복잡한 선택자도 모두 사용 가능

10. 실무에서의 활용

10.1 테마 전환 시스템

  • 다크 모드 / 라이트 모드
  • 모든 UI 요소를 일괄 변경
  • 사용자 경험 개선

10.2 대량 데이터 처리

  • 게시물 목록 스타일링
  • 상품 카드 디자인 변경
  • 메뉴 항목 일괄 수정

11. 성능 최적화 팁

11.1 변수에 저장하기

// 좋은 방법: 한 번만 선택

var alist = document.querySelectorAll('a');

while (i < alist.length) { ... }

// 나쁜 방법: 매번 선택

while (i < document.querySelectorAll('a').length) { ... }

11.2 효율적인 반복

  • 필요한 요소만 선택
  • 불필요한 반복 작업 최소화
  • 브라우저 성능 고려

12. 핵심 기억사항

  1. querySelectorAll(): 여러 요소를 배열로 선택
  2. while + length: 배열 크기에 맞춘 안전한 반복
  3. 동적 처리: 요소 개수에 관계없이 일괄 처리
  4. 확장성: 새로운 요소 추가 시 자동 반영
