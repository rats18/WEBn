1. 목표: 토글(Toggle) 기능 구현

  • 하나의 버튼으로 Night ↔ Day 모드 전환
  • 현재 상태에 따라 자동으로 반대 모드로 변경

2. 구현 과정

2.1 버튼 HTML 구조

<input type="button" value="night" id="night_day" onclick="...">

2.2 현재 버튼 값 확인 방법

document.querySelector('#night_day').value

// 결과: "night" 또는 "day"

2.3 조건문 구조

if (document.querySelector('#night_day').value === 'night') {

    // Night 모드로 변경하는 코드

    document.querySelector('body').style.backgroundColor = 'black';

    document.querySelector('body').style.color = 'white';

    document.querySelector('#night_day').value = 'day';

} else {

    // Day 모드로 변경하는 코드

    document.querySelector('body').style.backgroundColor = 'white';

    document.querySelector('body').style.color = 'black';

    document.querySelector('#night_day').value = 'night';

}

3. 핵심 로직

3.1 상태 변경 순서

  1. 현재 버튼 값 확인: night 또는 day
  2. 조건에 따른 동작:
    • 버튼이 night면 → 어두운 테마 적용 + 버튼을 day로 변경
    • 버튼이 day면 → 밝은 테마 적용 + 버튼을 night로 변경

3.2 동작 원리

// 처음 클릭 (버튼값: "night")

if ("night" === "night") // true

→ 어두운 테마 적용 + 버튼값을 "day"로 변경

// 두번째 클릭 (버튼값: "day")

if ("day" === "night") // false

→ else 구문 실행 → 밝은 테마 적용 + 버튼값을 "night"로 변경

4. 실습 체험

4.1 완성된 토글 기능

버튼을 여러 번 클릭해보세요! 클릭할 때마다 테마가 바뀌고 버튼 텍스트도 변경됩니다.

4.2 버튼 값 확인 체험

5. 조건문의 실용적 가치

5.1 문제 해결

  • 2개 버튼의 불편함 해결
  • 사용자가 상태 판단할 필요 없음
  • 더 세련된 인터페이스 제공

5.2 프로그래밍적 의미

  • 관념적 도구(비교연산자, Boolean, 조건문)가 현실적 문제 해결
  • 단순한 기능도 똑똑하게 동작하게 만듦

6. 핵심 개념

  1. 조건부 실행: 상황에 따라 다른 코드 실행
  2. 상태 관리: 버튼 값으로 현재 상태 추적
  3. 사용자 경험 개선: 더 직관적인 인터페이스

7. 2강 목표 달성

2강에서 제시했던 "하나의 버튼으로 Night & Day 모드를 전환하는 기능"이 완성됨. 비교연산자와 조건문을 활용해서 프로그램이 현재 상태를 스스로 판단하고 적절한 동작을 수행하는 지능적인 기능 구현.
