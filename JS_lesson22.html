1. 배열 vs 객체

1.1 배열의 특징

var coworkers = ['egoing', 'leezche'];

coworkers[0];  // 'egoing' (순서로 접근)

coworkers[1];  // 'leezche'

  • 순서대로 정보 저장
  • 인덱스(숫자)로 접근

1.2 객체의 특징

  • 순서 없이 정보 저장
  • 이름(key)으로 접근
  • 이름이 있는 정리정돈 상자

2. 객체 기본 문법

2.1 객체 생성 (Object Literal)

var coworkers = {

    "programmer": "egoing",

    "designer": "leezche"

};

  • 중괄호 {} 사용 (배열은 대괄호 [])
  • key: value 쌍으로 데이터 저장

2.2 데이터 읽기

// 점 표기법 (Dot Notation)

coworkers.programmer;  // "egoing"

coworkers.designer;    // "leezche"

// 대괄호 표기법 (Bracket Notation)

coworkers['programmer'];  // "egoing"

coworkers['designer'];    // "leezche"

5. 두 가지 접근 방법

5.1 점 표기법 (Dot Notation)

coworkers.programmer;

  • 간단하고 직관적
  • 공백이나 특수문자 불가

5.2 대괄호 표기법 (Bracket Notation)

coworkers['data scientist'];

  • 공백이나 특수문자 가능
  • 문자열로 key 지정

7. 핵심 개념

7.1 Key-Value 구조

  • Key: 데이터에 붙이는 이름표 (programmer, designer)
  • Value: 실제 데이터 (egoing, leezche)

7.2 Object Access Operator

  • 점(.): 객체에 접근하는 연산자
  • coworkers.programmer → coworkers 객체의 programmer 속성

7.3 유연한 데이터 관리

  • 순서 없음: 어떤 순서로든 데이터 저장/접근 가능
  • 의미있는 이름: 숫자 인덱스 대신 의미있는 key 사용

8. 배열 vs 객체 선택 기준

8.1 배열을 사용할 때

  • 순서가 중요한 데이터: 점수 목록, 단계별 과정
  • 같은 성격의 데이터: 과일 목록, 색깔 목록
  • 반복 처리가 주목적: 모든 항목에 동일한 작업

8.2 객체를 사용할 때

  • 이름으로 구분하는 데이터: 사람의 속성, 설정값
  • 다른 성격의 데이터: 이름, 나이, 직업 등
  • 특정 속성에 접근: person.name, car.brand

11. 객체의 한계와 다음 단계

객체 단독으로도 매우 유용하지만, 다음 강의에서 배울 객체의 반복 처리와 메서드를 결합하면 더욱 강력한 기능을 구현할 수 있습니다.

12. 핵심 가치

"객체는 이름표가 붙은 수납상자로, 순서가 아닌 의미있는 이름으로 데이터를 관리할 수 있게 해준다"

배열이 순서 중심의 데이터 관리라면, 객체는 의미 중심의 데이터 관리입니다. 이제 더욱 체계적이고 직관적인 데이터 구조를 활용할 수 있게 되었습니다!
